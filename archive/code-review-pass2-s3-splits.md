**Risks:** 1) Hot-reload surface shrinks — the real behavioral cost. reloadable.ts (17-21, 55) documents that reload depth is exactly one level: intent-v1.ts re-executes under a ?v query, but its STATIC imports resolve to already-cached modules. This is a documented dev-aid boundary, not product behavior, but it will surprise the owner mid-iteration — the plan updates the reloadable.ts doc, and the alternative (dynamic busted imports inside intent-v1.ts) is NOT behavior-preserving and is rejected.
